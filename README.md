# SAR Internal Wave Segmentation Pipeline

This repository contains a pipeline for preparing a dataset of
Sentinel-1 IW (Interferometric Wide swath) SAR images and training a
semantic segmentation model to detect **internal wave** signatures in
the ocean surface.

The pipeline takes you from raw manual annotations (LabelMe JSON files)
all the way to a PyTorch-ready `Dataset`/`DataLoader`, through four
scripts that are meant to be run **in order** (further updates will be made to this repository for adding all the other scripts to complete the pipeline of a full trained U-Net):

```
1. ConvJson.py       Annotations (JSON)  -> Binary masks (PNG)
2. Patches.py         Large images/masks  -> 256x256 patches
3. SeparateData.py    Patches             -> Train / Val / Test split
4. DataSet.py          Patches            -> PyTorch Dataset & DataLoader
```

---

## 1. `ConvJson.py` — Annotation to Mask Conversion

Converts LabelMe-style JSON annotations into binary segmentation masks.

- Reads every `.json` file in the annotations folder.
- Uses `imageHeight` / `imageWidth` from each JSON to create a black
  (all-zero) canvas the same size as the original image.
- Draws every `line`, `linestrip`, or `polygon` shape onto the canvas as
  a white (255) open polyline, 5 pixels thick.
- Saves one `.png` mask per JSON file, using the same base filename.

**Input:** `jsons_dir` (LabelMe `.json` files)
**Output:** `masks_output_dir` (binary `.png` masks)

> Note: shapes are drawn as *open outlines*, not filled regions — even
> polygon annotations are rendered as a traced path (`isClosed=False`),
> not a solid blob.

---

## 2. `Patches.py` — Patch Extraction

Slices large SAR images and their masks into fixed-size, non-overlapping
`256x256` patches so they can be fed to a segmentation model.

- Slides a non-overlapping window across each image/mask pair.
- Any patch whose mask contains **at least one wave pixel** is always
  saved.
- Patches that are **pure ocean** (mask is entirely zero) are only kept
  10% of the time (`chance_salvar_oceano_puro = 0.10`), to avoid
  flooding the dataset with uninformative background and keep classes
  reasonably balanced.
- A fixed random seed (`42`) makes the "which background patches survive"
  decision reproducible.

**Input:** `imagens_grandes_dir` / `mascaras_grandes_dir` (full-size images/masks)
**Output:** `imagens_patches_dir` / `mascaras_patches_dir` (256x256 patches)

Prints a summary at the end:
- number of wave patches saved
- number of pure-ocean patches saved
- total patches generated

---

## 3. `SeparateData.py` — Train / Validation / Test Split

Splits the patch collection generated by `Patches.py` into three
subsets by **moving** files on disk:

| Split      | Proportion | Destination folders            |
|------------|-----------:|---------------------------------|
| Training   | ~80%       | stays in `Waves` / `Masks`      |
| Validation | ~10%       | `ValWaves` / `ValMasks`         |
| Test       | ~10%       | `TestWaves` / `TestMasks`       |

- File list is shuffled once with a fixed seed (`42`) for a reproducible,
  random (not sequential) split.
- Every image patch is moved together with its correspondingly named
  mask patch, so image/mask pairs stay aligned across splits.

**Input/Output:** operates in-place on `data_dir` (the `Patches` folder),
moving files between its `Waves`/`Masks` subfolder and new
`Val*`/`Test*` subfolders.

> ⚠️ This script **moves**, not copies, files — running it twice on the
> same data will re-split whatever remains in the training folder.

---

## 4. `DataSet.py` — PyTorch Dataset & Augmentation

Defines the `Dataset` class and helper functions used to feed patches
into a PyTorch training loop.

### `Dataset(BaseDataset)`
- Loads an image (`cv2.imread`, converted BGR → RGB) and its matching
  mask (grayscale) by index.
- Binarizes the mask (`> 0` → `1.0`) and adds a channel dimension
  (`H, W` → `H, W, 1`).
- Applies `augmentation` (e.g. flips, rotation, brightness/contrast) and
  then `preprocessing` (e.g. normalization, tensor conversion), both
  optional and pluggable via [Albumentations](https://albumentations.ai/).

### Helper functions
- **`get_training_augmentation()`** — horizontal/vertical flips, 90°
  rotation, brightness/contrast jitter, then tensor conversion.
- **`get_validation_augmentation()`** — tensor conversion only, no
  random transforms, so evaluation stays deterministic.
- **`visualize(image, mask, label)`** — displays an image/mask pair
  side by side with `matplotlib`, handling both NumPy arrays and
  PyTorch tensors.

### Script mode (`__main__`)
When run directly, the script:
1. Builds `train_dataset` (with augmentation) and `valid_dataset`
   (without).
2. Wraps both in `DataLoader`s (`batch_size=8`).
3. Prints dataset/loader sizes.
4. Visualizes one sample from the training set as a sanity check.

**Expected folder layout** (under `data`):
```
Waves/       training images
Masks/       training masks
ValWaves/    validation images
ValMasks/    validation masks
TestWaves/   test images
TestMasks/   test masks
```


## Typical Workflow

```bash
# 1. Turn manual JSON annotations into binary masks
python ConvJson.py

# 2. Cut large images/masks into 256x256 patches
python Patches.py

# 3. Split patches into train/val/test folders
python SeparateData.py

# 4. Build the PyTorch Dataset/DataLoader and sanity-check a sample
python DataSet.py
```

## Notes & Caveats

- All directory paths in these scripts are currently **hard-coded**
  (Windows-style paths, e.g. `D:/Mestrado/ImagensSAR/...`). Update them
  at the top of each file before running on a different machine.
- Random seeds are fixed (`42`) in `Patches.py` and `SeparateData.py`
  for reproducibility — change or remove them if you want a different
  random split/sample each run.
- `SeparateData.py` moves files rather than copying them; keep a backup
  of your patch folders if you need to re-run the split.
